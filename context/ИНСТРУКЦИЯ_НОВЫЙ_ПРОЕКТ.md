# Инструкция: новый проект из KB_Шаблон

**Для агента.** Cursor rule: [`.cursor/rules/project-bootstrap.mdc`](../.cursor/rules/project-bootstrap.mdc).

---

## Для пользователя (кратко)

1. **Клонировать или скопировать** репозиторий шаблона в новую папку (не работать в материнском `KB_Шаблон` на GitHub).
2. **Cursor → Open Folder** → эта копия.
3. В чате одна фраза, например: **«Создай новый проект KB_MyApp»**.
4. Ответить на **Ask Question** (имя, путь, public/private GitHub).
5. После bootstrap — **переименовать папку** на диске, если Cursor ещё открыт со старым путём.

---

## Триггер

Пользователь в **копии** шаблона называет новое имя проекта (например `KB_MyApp`, «новый проект», «bootstrap»).

**Не триггер для материнского шаблона** — там только разработка самого шаблона.

---

## Алгоритм агента

### Шаг 0 — проверка

| Условие | Действие |
|---------|----------|
| cwd = материнский `KB_Шаблон` (путь из README) | **Стоп.** Объяснить: скопировать/клонировать. |
| `AGENTS.md` уже без `KB_Шаблон` | Проект уже bootstrapped — не повторять без запроса. |

### Шаг 1 — Ask Question

Форма с **тремя** вопросами; у **каждого** `"allow_multiple": true`:

1. Имя проекта (`KB_*`, латиница).
2. Путь папки (текущая / другой абсолютный путь).
3. GitHub: public / private / позже.

### Шаг 2 — команда

```powershell
py -3 tools/bootstrap_project.py --name <Имя> [--path <путь>] [--github-public | --github-private]
```

Опционально `--dry-run` по просьбе пользователя.

### Шаг 3 — действия скрипта

| # | Действие |
|---|----------|
| 1 | Замена `KB_Шаблон` → `<Имя>` в `.md`, `.mdc`, `AGENTS.md`, `README`, `context/*`, `project-domain.mdc` |
| 2 | Замена старого абсолютного пути → новый (`--path` или cwd) |
| 3 | `VERSION` → `1.0.0` |
| 4 | `КОНТЕКСТ_АГЕНТА.md` — стартовый (без v1.0.x шаблона) |
| 5 | `КОНТЕКСТ_ИДЕЙ.md` — пустой шаблон |
| 6 | `КОНТЕКСТ_ПОЛЬЗОВАТЕЛЯ.md` — prefs сохранены, заголовок с новым именем |
| 7 | Git: удалить `origin`; `git add -A`; commit `v1.0.0: база - инициализация <name>` |
| 8 | `gh repo create <name> --source=. --remote=origin --push` (если выбран GitHub) |

### Шаг 4 — ответ пользователю

- Имя проекта, VERSION `1.0.0`, хеш коммита.
- URL GitHub (если создан).
- Напоминание **переименовать папку**, если cwd ≠ целевой путь.

---

## Безопасность скрипта

- Refuse, если нет маркера: `VERSION` + `KB_Шаблон` в `AGENTS.md` (override: `BOOTSTRAP_OK=1`).
- Refuse в материнской папке `KB_Шаблон` (имя cwd или `--name KB_Шаблон`).
- Имя проекта — только `[A-Za-z0-9_-]+` (GitHub).

---

## Материнский шаблон на GitHub

Репозиторий шаблона — **источник для копий**. Его **не** bootstrap'ить.

Разработка bootstrap-функции — только в материнском `KB_Шаблон`; применение — только в копиях.
