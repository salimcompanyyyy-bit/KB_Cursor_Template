# Промпты для Cursor — перенос настроек агента в новый проект

**Как пользоваться:** откройте **новый проект** (или пустую папку) в Cursor → **новый чат** → вставляйте промпты **по порядку** (ПРОМПТ 0, 1, 2…). Дождитесь выполнения каждого шага перед следующим.

**Шаблон-источник (готовый):** `C:\Users\Kawa\Desktop\Кб\KB_Шаблон`  
**Запасной источник:** `C:\Users\Kawa\Desktop\Кб\KB_MIB_PARSER`

**Перед началом замените в промптах:**
- `<ПУТЬ>` — полный путь нового проекта, напр. `C:\Users\Kawa\Desktop\Кб\МойПроект`
- `<НАЗВАНИЕ>` — имя проекта, напр. `МойПроект`

---

## ПРОМПТ 0 — контекст (необязательно, если агент «с нуля»)

```text
Ты работаешь над новым проектом. Нужно развернуть дисциплину Cursor-агента как в KB_MIB_PARSER / KB_Шаблон:
- .cursor/rules/*.mdc (alwaysApply)
- Шаблон_Сохранения.md, AGENTS.md, VERSION, context/*
- auto-commit после порции (VERSION + git commit), Ask Question до новой функциональности
- русский язык, минимальный дифф

Источник для копирования: C:\Users\Kawa\Desktop\Кб\KB_Шаблон
Не копировать код MIB (mib/, tg_control_bot.py, Chrome, Excel COM).

Выполняй мои следующие промпты строго по порядку. После каждого шага — кратко отчитайся что создано.
```

## ПРОМПТ 1 — папка, git, .gitignore

```text
Шаг 1 переноса настроек агента.

1. Убедись что workspace = <ПУТЬ> (если папки нет — создай).
2. Выполни git init (если .git ещё нет).
3. Создай .gitignore:

__pycache__/
*.py[cod]
.env
.vscode/
reports/*
!reports/.gitkeep
.git_commit_msg*.txt

4. Создай пустую папку reports/ и файл reports/.gitkeep
5. git status — покажи результат. Коммит пока не делай.
```

## ПРОМПТ 2 — копирование starter pack

```text
Шаг 2: скопируй starter pack из шаблона в текущий проект.

Источник: C:\Users\Kawa\Desktop\Кб\KB_Шаблон
Назначение: <ПУТЬ> (текущий workspace)

Скопируй через PowerShell:

$Src = "C:\Users\Kawa\Desktop\Кб\KB_Шаблон"
$Dst = "<ПУТЬ>"

New-Item -ItemType Directory -Force -Path "$Dst\.cursor\rules", "$Dst\context\АРХИВ" | Out-Null

Copy-Item "$Src\Шаблон_Сохранения.md" $Dst
Copy-Item "$Src\AGENTS.md" $Dst -ErrorAction SilentlyContinue
Copy-Item "$Src\README.md" $Dst -ErrorAction SilentlyContinue
Copy-Item "$Src\context\КОНТЕКСТ_СОХРАНЕНИЕ.md" "$Dst\context\"
Copy-Item "$Src\context\ПЛАН_ПЕРЕНОС_НАСТРОЕК_CURSOR.md" "$Dst\context\"
Copy-Item "$Src\context\ПРОМПТЫ_ПЕРЕНОС_В_НОВЫЙ_ПРОЕКТ.md" "$Dst\context\" -ErrorAction SilentlyContinue

Get-ChildItem "$Src\.cursor\rules\*.mdc" | Where-Object { $_.Name -ne "mib-context.mdc" } | Copy-Item -Destination "$Dst\.cursor\rules\"

Если VERSION нет — создай файл VERSION со строкой 1.0.0

НЕ копируй: mib-context.mdc, историю КОНТЕКСТ_АГЕНТА.md из MIB, любой код mib/, data/, xlsx.

Покажи дерево файлов в корне проекта (dir / tree). Коммит не делай.
```

## ПРОМПТ 6 — первый коммит

```text
Шаг 6: первый git commit starter pack.

1. git status — убедись нет .env, xlsx, секретов
2. VERSION должен быть 1.0.0
3. git add .
4. Создай .git_commit_msg.txt (UTF-8):

v1.0.0: добавление - starter pack Cursor

1. Шаблон_Сохранения.md и .cursor/rules.
2. context/*, AGENTS.md, README для <НАЗВАНИЕ>.

5. git commit -F .git_commit_msg.txt
6. git log -1 --format="%h %s"

В ответе обязательно: v1.0.0 и хеш коммита.
```

## ПРОМПТ 7 — проверка агента (тестовая задача)

```text
Шаг 7 — проверка дисциплины.

Прочитай Шаблон_Сохранения.md, все .cursor/rules/, AGENTS.md, context/*.

Добавь в README.md один абзац «## Проверка агента» с текстом «Starter pack работает, дата: <сегодня>».

Затем по дисциплине:
- VERSION → 1.0.1
- обнови КОНТЕКСТ_АГЕНТА.md (запись v1.0.1)
- git commit с форматом v1.0.1: контекст - проверка readme
- в ответе: v1.0.1 и хеш

Это проверка что auto-commit работает.
```
