# План переноса настроек Cursor-агента в новый проект

**Статус:** утверждённый справочник (контекст-only).  
**Источник:** опыт KB_MIB_PARSER (v3.1.155, 01.06.2026).  
**Назначение:** один документ, по которому можно развернуть «такое же» поведение агента (коммиты, версии, вопросы, контекст) в **любом** новом репозитории.

---

## 0. Что именно переносим (словами)

«Настройки», из‑за которых агент работает хорошо в MIB — это **не одна галочка в Cursor**, а **стек из 4 слоёв**:

| Слой | Где живёт | Переносится в новый проект? |
|------|-----------|----------------------------|
| **A. User Rules** | Cursor → Settings → Rules (глобально) | Уже на все проекты; при необходимости дописать 1–2 строки |
| **B. Skills** | `%USERPROFILE%\.cursor\skills-cursor\` | Глобально; копировать не нужно |
| **C. Project rules** | `.cursor/rules/*.mdc` | **Да — копировать** (часть файлов) |
| **D. Контекст + шаблон** | `Шаблон_Сохранения.md`, `context/*`, `AGENTS.md`, `VERSION` | **Да — копировать / создать по шаблону** |
| **E. Режим Multitask** | UI Cursor (чат) | Не файл; включается вручную при необходимости |

**Не переносится:** история чатов, agent-transcripts, доменный код MIB (`mib/`, TG, Chrome 9222), специфика mib.uz.

---

## 1. Слой A — глобальные User Rules (Cursor)

Уже действуют на **все** проекты. Типичные темы (проверить в Settings → Rules):

- стиль ответов (русский, markdown, без лишнего bold);
- «реальная среда — запускай команды сам»;
- PR через `gh`;
- **общее правило:** «коммить только по просьбе».

### ⚠️ Конфликт с MIB

В **этом** репозитории проектные правила **перебивают** глобальное «коммить по просьбе»:

- `.cursor/rules/save-on-code-change.mdc` — коммит **обязателен** после порции кода;
- исключение: пользователь написал **«без коммита»** в текущем чате.

### Что добавить в User Rules для нового проекта (рекомендация)

Одна строка (если копируете MIB-дискiplину):

```text
В проектах с файлом Шаблон_Сохранения.md и .cursor/rules/save-on-code-change.mdc
приоритет у проектных правил: после порции кода — VERSION + git commit без ожидания слова «закоммить».
```

## 3. Слой C — `.cursor/rules/` (главное для копирования)

### 3.1. Копировать «как есть» (универсальные)

| Файл | `alwaysApply` | Суть |
|------|---------------|------|
| `mandatory-rules-read-first.mdc` | true | Сначала прочитать все `.mdc`, шаблон, контекст — потом код |
| `save-on-code-change.mdc` | true | VERSION + git commit после порции; формат ответа с хешем |
| `project-context-template.mdc` | true | Вопросы до кода, Ask Question, версии, словарь типов коммитов, обновление context/ |

### 3.2. Копировать только если нужна та же фича

| Файл | Когда копировать |
|------|------------------|
| `commits-export-format.mdc` | Если переносите `tools/export_commits_xlsx.py` и папку `reports/` |
| `mib-context.mdc` | **Не копировать** — заменить своим `project-domain.mdc` (см. §6) |

### 3.3. Структура frontmatter каждого `.mdc`

```yaml
---
description: Краткое описание (видно в rule picker)
alwaysApply: true          # true = всегда в контексте агента
# globs: **/*.py           # только для file-specific правил
---
```

Cursor автоматически подхватывает `.cursor/rules/` при открытии workspace.

## 4. Слой D — файлы репозитория

### 4.1. Обязательный минимум (starter pack)

```
новый-проект/
├── AGENTS.md                      # точка входа для агента
├── VERSION                        # одна строка, напр. 1.0.0
├── Шаблон_Сохранения.md           # ГЛАВНЫЙ документ дисциплины (скопировать из MIB)
├── .cursor/
│   └── rules/
│       ├── mandatory-rules-read-first.mdc
│       ├── save-on-code-change.mdc
│       └── project-context-template.mdc
└── context/
    ├── КОНТЕКСТ_СОХРАНЕНИЕ.md      # указатель на шаблон (скопировать)
    ├── КОНТЕКСТ_АГЕНТА.md           # создать заново (5–30 строк)
    ├── КОНТЕКСТ_ПОЛЬЗОВАТЕЛЯ.md     # prefs пользователя
    ├── КОНТЕКСТ_ИДЕЙ.md             # бэклог (пустой шаблон)
    └── АРХИВ/                       # готовые спеки, не смешивать с «живым» контекстом
```

### 4.2. Опционально (если нужен такой же UX, как в MIB)

| Арtefact | Путь | Зачем |
|----------|------|--------|
| Выгрузка коммитов | `tools/export_commits_xlsx.py` | Excel-отчёт по git log |
| Правило выгрузки | `.cursor/rules/commits-export-format.mdc` | Жёсткий формат 8 колонок |
| Reports | `reports/.gitkeep` + `.gitignore` → `reports/*` | Артеfacts не в git |

### 4.3. `AGENTS.md` — что править после копирования

1. Заменить ссылки на домен MIB → название нового проекта.
2. Убрать пункты про Excel COM / Chrome 9222 / mib.uz, если не актуально.
3. Оставить: порядок чтения правил, цикл VERSION → context → commit, Ask Question.

### 4.4. `context/КОНТЕКСТ_АГЕНТА.md` — стартовый шаблон

```markdown
# Контекст агента — <ИМЯ ПРОЕКТА>

**Версия:** см. корневой VERSION.
**Обновляется:** после каждой порции, меняющей поведение.

## Назначение проекта
(2–5 предложений)

## Стек
(язык, фреймворк, как запускать)

## Жёсткие правила
- См. Шаблон_Сохранения.md и .cursor/rules/
- Новая видимая функциональность → Ask Question до кода

## Сделано (кратко, последние 5–10 записей)
- v1.0.0 — инициализация starter pack
```

**Не копировать** историю из MIB `КОНТЕКСТ_АГЕНТА.md` — только структуру.

### 4.5. `context/КОНТЕКСТ_ПОЛЬЗОВАТЕЛЯ.md` — что перенести из MIB

| Pref | Значение в MIB | Перенести? |
|------|----------------|------------|
| Язык | русский | да |
| Ask Question | `allow_multiple: true` всегда | да |
| Коммиты | auto после порции (проектное правило) | да, если starter pack |
| Push | только по просьбе | да |
| Multitask | по желанию в чате | упомянуть |

### 4.6. `context/КОНТЕКСТ_ИДЕЙ.md` — шаблон

```markdown
# Контекст идей — только НЕ реализованное

Реализованное → КОНТЕКСТ_АГЕНТА.md; детали закрытых волн → context/АРХИВ/.

## Дорожная карта
| § | Приоритет | Статус | Тема |
|---|-----------|--------|------|
| §1 | P1 | ⬜ | … |
```

## 5. Поведение, которое даёт «как в MIB»

### 5.1. Авто-коммиты и VERSION

**Цикл порции (не менять порядок):**

1. `git status` (+ pytest при необходимости)
2. `VERSION` → +1 PATCH (MINOR/MAJOR только по явному согласию)
3. `context/КОНТЕКСТ_АГЕНТА.md` — если менялось поведение
4. `git add` (без секретов, без .gitignore-путей)
5. `git commit` — формат ниже
6. В ответе: **vX.Y.Z** + **хеш коммита**
7. `git push` — **только** по просьбе

**Формат коммита:**

```text
vX.Y.Z: <тип> - <кратко>

1. …
2. …
```

**Типы (одно слово):** `добавление`, `исправление`, `функция`, `рефакторинг`, `контекст`, `база`, `безопасность`, `производительность`.

**Windows / кириллица:** `git commit -F path\to\msg.txt` (UTF-8 файл).

### 5.2. Вопросы до кода + Ask Question

- Новая **видимая** функциональность → сначала вопросы, **без ответа код не писать**.
- Варианты известны → инструмент **Ask Question** в Cursor.
- Hard-rule MIB: у **каждого** вопроса `"allow_multiple": true` (даже если логически нужен один ответ — два последовательных вопроса).
- Уточняющие вопросы — **только** через Ask Question, не простым текстом (проектное правило MIB).

### 5.3. Multitask Mode

- Режим чата Cursor: агент делегирует тяжёлые задачи фоновым subagent.
- **Не настраивается файлами проекта.**
- Включать, когда нужны параллельные исследования / большие порции.

### 5.4. Минимальный дифф

User rule + `project-context-template.mdc`: менять только то, что нужно для задачи; не рефакторить «заодно».

## 8. PowerShell: быстрое копирование starter pack

**Замените** `$Src` и `$Dst` перед запуском.

```powershell
$Src = "C:\Users\Kawa\Desktop\Кб\KB_MIB_PARSER"
$Dst = "C:\Users\Kawa\Desktop\МойНовыйПроект"

New-Item -ItemType Directory -Force -Path "$Dst\.cursor\rules", "$Dst\context\АРХИВ" | Out-Null

Copy-Item "$Src\Шаблон_Сохранения.md" $Dst
Copy-Item "$Src\AGENTS.md" $Dst
Copy-Item "$Src\context\КОНТЕКСТ_СОХРАНЕНИЕ.md" "$Dst\context\"

@(
  "mandatory-rules-read-first.mdc",
  "save-on-code-change.mdc",
  "project-context-template.mdc"
) | ForEach-Object { Copy-Item "$Src\.cursor\rules\$_" "$Dst\.cursor\rules\" }

Set-Content -Path "$Dst\VERSION" -Value "1.0.0" -Encoding utf8
Write-Host "Starter pack copied to $Dst — edit AGENTS.md and create context/*.md"
```

После копирования **обязательно** отредактировать `AGENTS.md` и создать три файла контекста (§4.4–4.6).

## 10. Частые ошибки при переносе

| Ошибка | Как избежать |
|--------|----------------|
| Скопировали `mib-context.mdc` | Заменить на `project-domain.mdc` |
| Скопировали весь `КОНТЕКСТ_АГЕНТА.md` из MIB | Только шаблон; история MIB не нужна |
| Global rule «коммить по просьбе» блокирует auto-commit | Добавить приоритет шаблона в User Rules (§1) |
| Агент не видит rules | Файлы в `.cursor/rules/`, workspace = корень git |
| Кириллица в `git log` битая | `git commit -F` с UTF-8 |
| Ждут push после каждой порции | Push только по явной просьбе |

## 12. Краткий итог

1. **Скопировать starter pack** (§4.1 + §8).  
2. **Написать домен** (`project-domain.mdc` + `КОНТЕКСТ_АГЕНТА.md`).
