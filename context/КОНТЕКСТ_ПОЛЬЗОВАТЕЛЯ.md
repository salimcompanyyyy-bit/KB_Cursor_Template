## Глобальное правило Cursor (User Rules)

В **Cursor → Settings → Rules** глобально может действовать правило «коммить только по просьбе». В **этом** проекте его **перебивают** проектные правила: после порции кода — **VERSION + git commit** без ожидания слова «закоммить» (исключение: **«без коммита»** в текущем чате).

Рекомендуемая строка в User Rules (если копируете MIB-дискiplину):

```text
В проектах с файлом Шаблон_Сохранения.md и .cursor/rules/save-on-code-change.mdc
приоритет у проектных правил: после порции кода — VERSION + git commit без ожидания слова «закоммить».
```

## Git и VERSION

- Приоритет: **`Шаблон_Сохранения.md`** + `.cursor/rules/save-on-code-change.mdc`
- После порции правок: **VERSION + git commit** в той же сессии — **не ждать** «закоммить»
- Исключение: **«без коммита»** в текущем чате
- **Push** — только по явной просьбе
- Формат коммита: `vX.Y.Z: тип - кратко` + 1–3 пункта; кириллица через `git commit -F file.txt` (UTF-8)
