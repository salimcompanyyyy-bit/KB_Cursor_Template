## Ожидания к работе агента (задано пользователем)

0. **Всегда спрашивать**, если что-то **непонятно** или хочется **уточнить** — не продолжать писать код, пока не станет ясно, что именно нужно (можно несколько вопросов подряд).
0а. **Нет конкретного исправления:** если по задаче **неясно**, *какое именно* исправление в коде нужно (нет воспроизведения, эталонного поведения, места в проекте или сценария) — **сначала задать вопросы**; не подбирать патч и не «чинить вслепую» до ответа.
1. **Сначала уточнения — потом выполнение.** Не начинать правки или «делать что-то непонятное» без вопросов, если задача не однозначна.
2. Уточняющих вопросов может быть **несколько**, пока не станет ясно, что именно делать.
2а. **Новая фича, формат логов/вывода, заметное поведение для пользователя:** не внедрять сразу — **сначала уточнить**; если варианты ответа **можно перечислить заранее** — задать вопрос через **форму Cursor** (**Ask Question** / вопрос с вариантами), чтобы не гадать и не переделывать.
2б. **Любые** уточняющие вопросы пользователю — **только** через **Ask Question** в Cursor (не списком в обычном сообщении чата). В формах **Ask Question** у **каждого** вопроса в JSON **всегда** **`"allow_multiple": true`** — **hard-rule** (требование пользователя **2026-05-26**), **без исключений**. **Никаких** `false` — даже для «ровно один из взаимоисключающих», «выберите одно», «один основной режим», «Другое». Пользователь **должен** иметь возможность отметить несколько вариантов сразу (в т.ч. несмежные — B и D). **Не полагаться на дефолт инструмента** — `allow_multiple: true` пишется явно в каждом вопросе. Перед отправкой формы — проверить JSON: **у всех** вопросов стоит `true`.
2в. **Запрещено** делать **radio** (single-select) **в любых** уточняющих вопросах. Если по логике задачи действительно нужен ровно один ответ — задать **два последовательных вопроса** (например: «что делать по умолчанию» / «что разрешено **дополнительно**») или прописать в тексте вопроса «отметьте только нужное», но `allow_multiple` остаётся `true`. **Запрещено** оправдывать single-select формулировками «нужен один основной режим» / «выберите одно» / «ровно один вариант» — это **не** основание отключать мультивыбор.

### Жёстко: вопросы до кода (явное требование пользователя)

- **Нельзя** переходить к правкам кода **новой видимой функциональности** (сообщения в консоли, формат и число строк лога, тексты, новые CLI-флаги, изменение порядка шагов для пользователя), пока **не заданы уточняющие вопросы** через **Ask Question** и **не получен ответ**.
- Если варианты ответа **можно сформулировать заранее** — **обязательно** **Ask Question** в Cursor, а не выбирать вариант «как кажется логичным» самому и **не** дублировать те же вопросы текстом в чате.
- В формах **Ask Question**: **`allow_multiple: true`** у **каждого** вопроса — **всегда**, **без исключений** (п. **2б**, **2в**). Любой `radio` запрещён. Требование пользователя **2026-05-26**: **все** уточняющие вопросы — **только** с мультивыбором.
- Наличие п. 0–2а выше **не заменяет** выполнение: правило действует **на каждый** такой запуск задачи. **«Забыл спросить» = нарушение ожиданий пользователя.**

3. **Язык ответов:** русский; **английские термины** (имена флагов, ошибок, API) — **с кратким пояснением по-русски**, если они неочевидны.
4. **Не нравится:** когда ассистент **делает действия без уточнения**, хотя намерение пользователя неочевидно (в т.ч. детали UI консоли, числа строк лога, тексты).
5. **После правок:** `context/КОНТЕКСТ_АГЕНТА.md` и при необходимости `context/КОНТЕКСТ_ПОЛЬЗОВАТЕЛЯ.md` обновлять **автоматически** (без отдельной просьбы «запиши в контекст»), если изменилось поведение или договорённости.
6. **Минимальный дифф** по коду сохраняется: не раздувать задачу без запроса.
7. **Git и версии:** после **существенных** правок ассистент **сам** обновляет **`VERSION`**, делает **`git add`** / **`git commit`** (без push, если пользователь не просил), сообщение — по формату **`Шаблон_Сохранения.md`**.
