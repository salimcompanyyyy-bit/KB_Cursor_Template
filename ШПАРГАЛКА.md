# Шпаргалка — KB_Шаблон

Кратко: **что где** и **как завести новый проект**.

---

## Две роли папки

| Роль | Папка | GitHub | Что делать |
|------|--------|--------|------------|
| **Материнский шаблон** | `KB_Шаблон` (локально) | [KB_Cursor_Template](https://github.com/salimcompanyyyy-bit/KB_Cursor_Template) | Только правки самого шаблона. **Не** писать «Создай новый проект» здесь — bootstrap заблокирован. |
| **Твой рабочий проект** | Любая **копия** (например `KB_MyApp`) | Новый repo после bootstrap | Обычная работа: код, задачи, коммиты. |

**Одной фразой:** шаблон на GitHub → копия → в Cursor говоришь имя → свой проект и свой GitHub.

---

## Новый проект — по шагам

### Шаг 1. Получить копию шаблона

#### Способ А — Git (рекомендуется)

```powershell
cd "C:\Users\Kawa\Desktop\Кб"
git clone https://github.com/salimcompanyyyy-bit/KB_Cursor_Template.git KB_MyApp
```

- `KB_MyApp` — **имя новой папки** (латиница: буквы, цифры, `_`, `-`).
- Появится: `C:\Users\Kawa\Desktop\Кб\KB_MyApp`.

#### Способ Б — копия в проводнике

1. Открыть `C:\Users\Kawa\Desktop\Кб\KB_Шаблон`
2. **Ctrl+C** → вставить в `Кб` → **Ctrl+V**
3. Переименовать копию, например `KB_MyApp`

> **Важно:** копия должна называться **не** `KB_Шаблон`.

### Шаг 2. Открыть в Cursor

**File → Open Folder** → папка **копии** (`KB_MyApp`), не материнский `KB_Шаблон`.

**Проверка:** внутри есть `AGENTS.md`, `VERSION`, `tools/bootstrap_project.py`.

### Шаг 3. Одна фраза в чате

```
Создай новый проект KB_MyApp
```

Имя в фразе = имя папки = будущий repo на GitHub.

### Шаг 4. Агент сам

- задаст **Ask Question** (имя, public/private GitHub);
- переименует `KB_Шаблон` → твоё имя во всех файлах;
- сбросит `VERSION` на **1.0.0** и context;
- создаст **новый** репозиторий на GitHub и **push**.

### Шаг 5. Папка на диске (если нужно)

Cursor не переименует открытую папку. Если после clone имя не то — закрой Cursor, переименуй папку в проводнике, открой снова.

## Если `git clone` не работает

| Ошибка | Решение |
|--------|---------|
| `git` не найден | [Git for Windows](https://git-scm.com/download/win) или **Способ Б** (копия в проводнике) |
| `repository not found` | Repo public: https://github.com/salimcompanyyyy-bit/KB_Cursor_Template — или `gh auth login` |

## Обычная работа в проекте (после bootstrap)

1. Открыть **свою** папку в Cursor.
2. Пишешь задачу — агент читает `AGENTS.md`, rules, `context/*`.
3. После порции правок — **VERSION + git commit** (без слова «закоммить»).
4. Новая фича — сначала **Ask Question** (`allow_multiple: true` всегда).
5. **Push** — только если сам попросишь.

## Полезные команды

**Зависимости (выгрузка коммитов):**

```powershell
pip install -r requirements.txt
```

**Коммиты за день в Excel:**

```powershell
py -3 tools/export_commits_xlsx.py --since 2026-06-02 --until 2026-06-02
```

**Bootstrap вручную (если агент не запустил):**

```powershell
py -3 tools/bootstrap_project.py --name KB_MyApp --github-public
```

## Cursor Settings (один раз на все проекты)

**Settings → Rules** — добавить, чтобы проектные коммиты не блокировались глобальным «коммить по просьбе»:

```text
В проектах с файлом Шаблон_Сохранения.md и .cursor/rules/save-on-code-change.mdc
приоритет у проектных правил: после порции кода — VERSION + git commit без ожидания слова «закоммить».
```
